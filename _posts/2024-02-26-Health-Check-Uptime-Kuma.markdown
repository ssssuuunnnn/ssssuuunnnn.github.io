最近因為需要維護的系統及API日益增多，並且內部的使用者也越來越多，在每日也無法顧及所有的系統及API的服務都是正常在執行，並且對於系統有問連線上的問題發上，需要有即時發現的能力，因此最近導入了這套服務「Uptime Kuma」。

而這套服務安裝非常的簡單，僅需要

「docker run -d — restart=always -p **3001**:3001 -v **uptime-kuma**:/app/data — name **uptime-kuma** louislam/uptime-kuma:1」

即可。

在這個容器化的世道上，一切什麼都包成docker很快就可以解決了。

安裝完後，可以看到系統的基本Dashboard

這套服務不僅可以進行網站、API的監控，還可以監看資料庫、Docker、Ping、Redis等等…，基本上可以連上的都可以。

介面上很易懂，在各個要監控的服務也可以進行分類、標籤以及群組，可以讓使用者容易的整理所需監看的服務。

他基本的作法就是在你指定的頻率去呼叫相對應的程式(服務)，時間上可以自由設定，如果遇到錯誤時也可以設定重試幾次。

當如果遇到錯誤時，也可以進行多種服務的通知，比如：Line、Telegram、Slack等等的，很多，有很多我也是沒有聽過。

基本上進入這個服務需要進行帳號密碼的登入。另外，它也可以讓使用者自行產生不用登入的監看頁面，所以相對是蠻方便的。

### 結論

在進行多個服務的監控時，有這樣的軟體可以很快的幫助管理者去知道系統目前是否正常運行，而有遇到錯誤時也可以馬上進行通知，這樣一來就不會發生User比管理者還早發現系統錯誤的情況，來增加User對管理者的信心。

並且本套系統在建立的時候也非常的簡單，操作介面也易懂，基本上初階的管理人員也可能容易上手，所以也是非常建議大家可以試試看。
