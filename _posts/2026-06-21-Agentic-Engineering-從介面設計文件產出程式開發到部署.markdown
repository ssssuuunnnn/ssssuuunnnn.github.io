最近比較有一點時間，就來整體目前實作執行 Agentic Engineering 的資料，這次在這邊跟大家分享到目前為止在做 Agentic Engineering 的一些心得，這次用的是 gemini 產出設計跟需求文件(PRD)，最後 copilot CLI 進行開發。

![](https://cdn-images-1.medium.com/max/1024/1*qXGNy9jwJkLiFQ-748USsg.png)

在目前主要分幾個流程，如下：

0.建立 PRD 規範 (gemini GEM)、開好 github repo. & pages  
1.桃園資料開放平台取資料  
2.資料給 gemini 進行介面規劃及介面說明  
3.需求及介面資料提供給 GEM 產生 PRD  
4.PRD 資料提供給 copilot cli  
5.本機介面測試  
6.自動部屬至 github 及 pages

範例結果：<https://ssssuuunnnn.github.io/Taoyuan_Housing_Transaction_Dashboard/>

對於文字所畫出來的線稿總感覺不夠具體，所以再次使用 nano banana 把線稿畫成圖檔，這樣看起來就具體多了。

![](https://cdn-images-1.medium.com/max/1024/1*z5BVpXI9MGJ7wH3Tf5c4DQ.png)

第二，產生 PRD，在此之前己經有建立好 gemini GEM， 己經把相關規範跟內容都提供給了 GEM，所以再提供功能說明、介面的設計、串接資料、技術棧、git 位置…等等。接下來就是確認文件內容是否與想法有出入，再進行調整。

第三，將 PRD 提供給 copilot CLI

這部很簡單，直接執行即可，因為 copilot CLI 會直接依照 PRD 的內容進行開發，要做的就是等待，喝個咖啡。

最後，只需要確認內容及功能無誤，就可以讓 copilot CLI 幫你把程式碼都推上 git 並產生好網頁，結果如下。

![](https://cdn-images-1.medium.com/max/1024/1*ChVCjXEmvmqBjg2erHlJ4w.png)

這次比較完整的分享了，透過較符合軟體開發流程的方式，進行網頁的開發。發現到，思維改為 Agentic Engineering ，在整體都是可控的情況下去開發，把 AI 真正做為一個協作的角色，而不是採用憑感覺(Vibe)的方式進行開發。

未來軟體工程師只要確認介面及功能(API)的正確性即可，而在產生 PRD 的部分(把話說清楚)就會變的更加重要，但在工程師方面，也需要對相關技術有更多的學習跟了解才行。
